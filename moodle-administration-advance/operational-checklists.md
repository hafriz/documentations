# Operational Checklists

## Weekly Operations

- Review scheduled-task and ad-hoc-task failures.
- Check backup completion and restore-test status.
- Review disk usage for Moodle application files, database storage, logs, and `moodledata`.
- Inspect authentication, enrolment, and integration errors.
- Review security announcements and plugin update notices.

## Monthly Operations

- Validate monitoring thresholds and alert routing.
- Review administrator accounts and privileged roles.
- Confirm documentation for integrations and service accounts.
- Test a sample restore in a non-production environment.
- Review performance trends and capacity forecasts.

## Before Major Changes

- Confirm business owner approval.
- Document the change plan, validation plan, and rollback criteria.
- Back up application files, database, and `moodledata` as needed.
- Rehearse in staging when the risk is significant.
- Communicate outage windows and user impact.
