# Plugins, Themes, and Site Configuration

## Why This Module Matters to You

This module helps you perform plugins, themes, and site configuration tasks in a way that is consistent, evidence-based, and safe for live Moodle users. The focus is on what you need to do, what to check, and how to explain your decisions.

## What You Will Be Able to Do

- Review plugin and theme requests from an operational risk perspective.
- Identify configuration changes that require testing or approval.
- Record before-and-after settings for safer rollback.
- Communicate visible site changes to affected users.

## Topics You Will Practice

- Plugin request intake.
- Compatibility and support ownership.
- Theme and branding impact.
- Site configuration review.
- Change validation and rollback planning.

## Hands-On Lab

Evaluate a plugin or theme request and decide whether it should be tested, deferred, or rejected with reasons.

## Your Evidence of Completion

By the end of this module, you should have:

- A completed checklist or decision note.
- Screenshots or written observations from a non-production Moodle site where available.
- A short explanation of the risk, validation method, and next action.

## Key Takeaways

- Intermediate administration is about repeatable workflows, not just knowing where menus are.
- Every meaningful change should have an owner, rationale, validation method, and rollback path.
- Logs, reports, and stakeholder confirmation should be used together before declaring an issue resolved.
