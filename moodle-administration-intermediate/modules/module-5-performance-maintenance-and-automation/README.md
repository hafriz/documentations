# Performance, Maintenance, and Automation

## Why This Module Matters to You

This module helps you perform performance, maintenance, and automation tasks in a way that is consistent, evidence-based, and safe for live Moodle users. The focus is on what you need to do, what to check, and how to explain your decisions.

## What You Will Be Able to Do

- Run routine site health checks before problems become incidents.
- Explain how cron, scheduled tasks, caches, backups, and storage affect user experience.
- Use simple checklists or scripts responsibly under local policy.
- Know when to escalate to senior system or infrastructure teams.

## Topics You Will Practice

- Cron and scheduled task awareness.
- Cache clearing and safe timing.
- Backup and storage checks.
- Basic performance symptoms.
- Lightweight automation opportunities.

## Hands-On Lab

Complete a monthly health checklist and identify which findings require immediate action, monitoring, or escalation.

## Your Evidence of Completion

By the end of this module, you should have:

- A completed checklist or decision note.
- Screenshots or written observations from a non-production Moodle site where available.
- A short explanation of the risk, validation method, and next action.

## Key Takeaways

- Intermediate administration is about repeatable workflows, not just knowing where menus are.
- Every meaningful change should have an owner, rationale, validation method, and rollback path.
- Logs, reports, and stakeholder confirmation should be used together before declaring an issue resolved.
