# Course and User Operations

## Why This Module Matters to You

This module helps you perform course and user operations tasks in a way that is consistent, evidence-based, and safe for live Moodle users. The focus is on what you need to do, what to check, and how to explain your decisions.

## What You Will Be Able to Do

- Prepare course categories and shells for a new teaching period.
- Validate user status, course visibility, and teacher access.
- Use course backup, restore, reset, and import options safely.
- Document a course readiness review for stakeholders.

## Topics You Will Practice

- Course lifecycle planning.
- Course templates and rollover checks.
- Bulk user and course operations.
- Visibility, dates, completion, and gradebook readiness.
- Validation evidence before release.

## Hands-On Lab

Review a sample course category, identify readiness issues, correct safe items in a practice site, and write a short sign-off note.

## Your Evidence of Completion

By the end of this module, you should have:

- A completed checklist or decision note.
- Screenshots or written observations from a non-production Moodle site where available.
- A short explanation of the risk, validation method, and next action.

## Key Takeaways

- Intermediate administration is about repeatable workflows, not just knowing where menus are.
- Every meaningful change should have an owner, rationale, validation method, and rollback path.
- Logs, reports, and stakeholder confirmation should be used together before declaring an issue resolved.
